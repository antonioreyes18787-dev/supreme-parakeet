What this prototype proves — and what it fixes

The five things it does

  • Answers basic questions from a knowledge base synced out of Google Drive + learned memory.
  • Codes every maintenance call automatically by category and priority.
  • Routes it to the correct property manager's email.
  • Pools all work orders in a back-end queue operators can track to resolution.
  • Learns — questions it can't answer are escalated, and once an operator answers, Sammy handles it on its own next time.

Built to fix what Dialpad & Vendoroo get wrong

Dialpad pain points → fixed
  • Per-minute & surprise inbound billing → one flat, predictable system.
  • Best AI locked in $150/user tiers → core AI built in, not gated.
  • Weak contact handling → callers tied to property & member records.
  • Phone and contact-center sold separately → one unified line.
Vendoroo pain points → fixed
  • Black-box AI → operators see and edit every answer & route.
  • Context handoff failures → structured, coded work orders.
  • No way to correct it → built-in teach-and-learn loop.
  • Generic vendor coordination → routed to your named PMs.

How the real version connects

Sign-in uses Google Workspace SSO (your @sam.org accounts), so anyone authorized can reach it from any device. The knowledge base syncs from a shared Google Drive folder; routing uses Gmail; and a telephony layer (e.g. a cloud number with speech-to-text) feeds calls into the same engine you see here.
